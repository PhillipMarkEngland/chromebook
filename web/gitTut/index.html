How to use Git and Github

What is Git?

Git is a version control system. This allows many developers to work on the same project while being on different networks.

Git will track every change and version of a project.

This allows you to view previous versions of a code and retrieve earlier versions of files.

You can "commit" your code to git and this will track and save the current state of your files.

Once you commit, you can then pull those files on a different machine.

Basic Git Commands

"git config --global user.name 'enter name here'" will set your name in git.

"git config --globql user.email 'enter email here'" will set your email in git.

"git init" will initialize a local git repository. If this command is ran within a directory, the entire directory will be initialized as a git repository.

"git add " will add files to an index

If you want to add all the files in a directory to the staging area, you can type "git add ."

If you want to add all files of a specific type to the staging area, just type "git add *."

If you modify a file while it is in the staging area, git will flag you and remind you to recommit the file to update the changes.

"git rm --cached " will remove a file from the stating area in the case that you do not want to commit a specific file.

"git status" will check files in the index

"git commit" will commit changes to the index

When you make a commitment to git, you will need to add a comment to communicate the changes you have made to the project. Running "git commit" will open up a text editor. Nano is the default text editor that will be used. If you want to change the editor used, just type "git config --global core.editor " This will make git use your prefered editor be default.

To bypass opening an editor to add a comment, just run "git commit -m ''"

"git push" will push files to a remote repository like github.

"git pull" will pull the latest version from a remote repository like github

"git clone" will clone a repository into a new directory.

"git --version" will let you see which version of git you a running. This is also a great way to check if you have git installed on your machine.

Git Bash

Git bash is a Windows utility that allows you to use a linux-like shell in windows. This allows windows users to maintain a linuxy feel when they are developing on a windows machine.

Using Git Ignore

Create a file within you git project called ".gitignore"

.gitignore functions very much like a text file

To ignore a file during commitment, add the name of file in plain text to this .gitignore file.

You can also included entire directories to .gitignore

Simply put the path to the directory you want to ignore within the .gitignore file.

Using Branches

Branches are used while pieces of a project are being developed.

Lets say you are working on creating a new navbar for a website, but you are not done with the nav bar. Creating a branch will allow you to work and make commitments to a "branch" of the project, without commiting the changes to the main brach. Once you are completed with your nav bar, you can then merge the changes to the main branch of the project.

Use "git branch " to add a new branch to a project.

To switch to a different branch run, "git checkout "

The name of the main branch is always called "master"

When you are ready to merge your changes to the master branch, go to the master branch and run "git merge "

Pushing Projects to Github

Go to github.com and add a new repository. The website will then provide step by step instructions on how to get your files added to the repository.
